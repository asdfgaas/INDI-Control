# 无动力滑翔飞行器纵向/简化侧向跟踪仿真说明

## 1. 项目概述

本项目给出一份可直接运行的 MATLAB 脚本，用于完成无动力滑翔飞行器的纵向/简化侧向跟踪仿真。脚本文件为：

- `glider_tracking_sim.m`

该脚本重点验证以下两类指令的跟踪能力：

- 恒定速度指令 `V_ref`
- 恒定法向加速度指令 `a_n_ref`

建模采用三维质点运动模型，只保留飞行器平动运动及其与速度、航迹角、航向角、位置相关的状态；不建立刚体转动动力学，不积分 `p/q/r` 的刚体角速度方程，也不继续建立舵机、舵偏分配与力矩方程。

因此，这个模型适合用于：

- 早期控制律方案验证
- BTT 思想下的简化纵向跟踪分析
- “速度通道 + 法向加速度通道”耦合机理演示
- 后续扩展到更完整 6-DOF 模型前的预研和参数整定

---

## 2. 建模思路

### 2.1 动力学模型

脚本采用无动力滑翔飞行器的三维质点模型，推力固定为：

- `T = 0`

状态量定义为：

- `x, y, z`：位置
- `V`：速度
- `gamma`：航迹倾角
- `psi`：航向角
- `theta`：俯仰角
- `phi`：滚转角

其中 `z` 采用“高度向上为正”的定义。

平动方程采用：

```matlab
x_dot = V * cos(gamma) * cos(psi)
y_dot = V * cos(gamma) * sin(psi)
z_dot = V * sin(gamma)
V_dot = -D / m - g * sin(gamma)
gamma_dot = (L * cos(phi) / m - g * cos(gamma)) / V
psi_dot = (L * sin(phi) / (m * V * cos(gamma)))
```

其中升力、阻力采用简化气动模型：

```matlab
L  = 0.5 * rho * V^2 * S * CL
D  = 0.5 * rho * V^2 * S * CD
CL = CL0 + CLa * alpha
CD = CD0 + K * CL^2
```

攻角采用近似关系：

```matlab
alpha = theta - gamma
```

空气密度采用简化指数模型：

```matlab
rho = rho0 * exp(-z / h_scale)
```

---

### 2.2 控制框架

控制结构采用“法向加速度基准 + 速度修正”的简化 BTT 风格框架。

控制逻辑如下：

1. 根据法向加速度指令反求所需升力：

```matlab
L_cmd = m * (a_n_ref + g * cos(gamma)) / cos(phi)
```

2. 根据升力需求反求升力系数：

```matlab
CL_cmd = 2 * L_cmd / (rho * V^2 * S)
```

3. 根据线性升力模型反求基准攻角：

```matlab
alpha_n = (CL_cmd - CL0) / CLa
```

4. 根据速度误差加入攻角修正项：

```matlab
delta_alpha_V = kV * (V - V_ref)
```

这里采用的符号约定是：

- 当 `V > V_ref` 时，增大攻角，提高阻力，使速度回落
- 当 `V < V_ref` 时，减小攻角，减小阻力，减缓掉速

5. 得到最终攻角指令：

```matlab
alpha_c = alpha_n + delta_alpha_V
```

6. 经过限幅后，转换为俯仰角指令：

```matlab
theta_c = alpha_c + gamma
```

---

### 2.3 内环简化处理

脚本不建立刚体转动动力学，而是直接用理想一阶环节来近似姿态内环：

```matlab
theta_dot = (theta_c - theta) / tau_theta
phi_dot   = (phi_c - phi) / tau_phi
```

这等价于假设：

- 姿态环足够快
- 角速度内环足够理想
- 不继续建立舵面和刚体转动方程

在后处理模块中，脚本还给出了等效的角速度指令量：

- `q_c = (theta_c - theta) / tau_theta`
- `p_c = (phi_c - phi) / tau_phi`
- `r_c = 0`

因此，这个模型是“控制做到角速度环附近”的简化实现。

## 3. 脚本结构说明

`glider_tracking_sim.m` 主要分为以下几个模块：

### 3.1 `Parameter Definition`

集中定义：

- 质量、面积、重力、空气密度等基础参数
- 升阻力模型参数
- 控制器参数
- 攻角、姿态等限幅
- 速度指令和法向加速度指令

这部分参数都集中在脚本开头，便于统一调整。

---

### 3.2 `Initial Condition`

定义仿真初始状态，包括：

- 初始高度
- 初始速度
- 初始航迹角
- 初始航向角
- 初始俯仰角
- 初始滚转角
- 初始位置

其中初始俯仰角由近似配平攻角给出：

```matlab
theta0 = gamma0 + alpha_trim
```

---

### 3.3 `Scenario Definition`

脚本默认包含两组场景：

- `With speed correction`
  启用速度修正项，用于验证速度指令和法向加速度指令联合跟踪

- `Without speed correction`
  关闭速度修正项，只保留法向加速度控制，用作对比

这两组场景有助于展示：

- 速度反馈项加入后，速度偏差是否减小
- 对法向加速度跟踪会产生怎样的耦合影响

---

### 3.4 `Numerical Integration`

采用 MATLAB 自带的 `ode45` 对状态方程进行数值积分，并设置了：

- `RelTol`
- `AbsTol`
- 终止事件 `stopEvents`

终止条件包括：

- 高度降到地面
- 速度过低，避免数值发散或失速附近异常

---

### 3.5 `pointMassModel`

该函数是仿真的核心，负责：

- 计算空气密度
- 计算当前攻角与气动力
- 计算实际法向加速度
- 由指令反求 `alpha_n`
- 加入速度修正得到 `alpha_c`
- 通过姿态一阶环节更新 `theta`、`phi`
- 返回完整状态导数

---

### 3.6 `postProcess`

用于在积分结束后统一计算绘图和分析所需量，包括：

- `alpha`
- `alpha_c`
- `alpha_n`
- `a_n`
- `CL`
- `CD`
- `rho`
- `q_c, p_c, r_c`

这样可以把“状态量”和“中间控制量”清晰区分开。

---

### 3.7 绘图模块

脚本默认绘制以下图像：

- 速度 `V` 与速度指令 `V_ref`
- 法向加速度 `a_n` 与指令 `a_n_ref`
- 攻角 `alpha` 与攻角指令 `alpha_c`
- 俯仰角 `theta`、航迹角 `gamma`
- 高度 `z`
- 纵向轨迹 `x-z`
- 滚转角 `phi`
- 三维轨迹图

## 5. 当前默认参数与说明

为了兼顾无动力条件下的可跟踪性与图像可读性，脚本当前采用了一组较稳健的默认参数：

- `V_ref = 195 m/s`
- `a_n_ref = 3.5 m/s^2`

这是经过实际试跑后调整得到的更稳定参数组。

原因是：

- 对无动力滑翔器而言，若同时要求较高法向加速度和较高速度保持，物理上会更难兼顾
- 当 `a_n_ref` 过大时，需要更大的升力，通常意味着更大的攻角和阻力
- 阻力增大后速度更容易下降，进而进一步推高所需升力，形成更强耦合

如果直接采用更激进的组合，例如：

- `V_ref = 230 m/s`
- `a_n_ref = 5 m/s^2`

则更容易出现：

- 攻角接近饱和
- 速度明显下降
- 跟踪误差增大

这并不是代码错误，而是无动力条件下控制目标之间存在明显物理约束。

## 6. 仿真结果应如何理解

### 6.1 主场景

主场景中同时启用：

- 法向加速度指令通道
- 速度修正通道

因此应重点观察：

- 速度曲线是否更接近 `V_ref`
- 法向加速度是否能基本贴近 `a_n_ref`
- 攻角是否保持在限幅范围内
- 高度是否持续下降，符合无动力滑翔特性

---

### 6.2 对比场景

对比场景中关闭速度修正项，仅保留法向加速度跟踪。

这个场景的主要用途是说明：

- 如果只按法向加速度需求调节攻角，速度通常会出现更明显漂移
- 加入速度修正后，速度保持能力会更好
- 但速度通道和法向加速度通道之间存在耦合，因此两个指标通常不能完全同时“零误差”

---

## 7. 数值保护与鲁棒性设计

脚本中加入了若干必要保护，用于避免数值计算异常：

- 对 `alpha_c` 做限幅
- 对 `theta_c`、`phi_c` 做限幅
- 对 `V` 设置最小阈值 `V_min`
- 对 `cos(gamma)` 设置下界，避免 `psi_dot` 分母过小
- 对 `cos(phi)` 设置下界，避免反求升力时除零
- 设置终止事件，避免飞行器“穿地”或速度过低后继续积分

这些保护的目的主要是：

- 保证仿真可稳定运行
- 提高参数扫描时的鲁棒性
- 方便后续继续扩展控制器和模型复杂度

---

## 8. 建议的后续扩展方向

如果后续要继续完善模型，可以考虑以下几个方向：

- 将 `phi_c` 设计成航向/横向轨迹控制量，而不是固定为零
- 增加侧向加速度或航向角跟踪任务
- 用更真实的升力、阻力、攻角限制模型
- 加入动态压强约束或过载约束
- 用二阶内环近似代替当前一阶姿态环
- 进一步扩展到完整刚体 6-DOF 动力学
- 加入舵机模型、舵偏分配与角速率闭环

## 10. 结论

本脚本实现了一个结构清晰、可直接运行的无动力滑翔飞行器跟踪仿真框架，满足以下特点：

- 使用三维质点模型
- 不包含刚体转动动力学
- 控制设计到姿态/角速度环附近
- 能同时验证恒定速度指令与恒定法向加速度指令跟踪
- 提供“有无速度修正项”的对比仿真
- 参数集中、注释清楚、便于后续扩展

若将其用于课程作业、方案预研或论文附录，建议配合脚本输出的图形一起说明：

- 无动力条件下速度保持与法向加速度保持之间存在天然耦合
- 更高的法向加速度指令会显著增加跟踪难度
- 当前脚本适合用于控制思想验证，而不是高保真工程定型
